Our product is a web application for users to find roommates

Target users are mainly college students and young professionals

Check Our Product

Functionalities

Sign Up

From here the user fills in basic information and opens an account

Sign In

From here the user logins to the app to find roommates match

Log Out

From here the user exits the account and returns to home page

Search

Users can fill in the preferred location, price range, lease term and pet preference to get matched users.

Profile

Users can edit the personal information, room preference and user avatar.

Chat

Users can contact each other through our built-in chat box. They are also able to view the chat history and delete any of them.

Admin Assistence

Our developers can login the admin account for the upkeep, configration and other reliable operations of our system

Technologies Integrated

Active Storage

It facilitates uploading user avatars to local storage or a cloud storage service and attaching those images to Active Record objects. It comes with a local disk-based service for development and testing and supports mirroring files to subordinate services for backups and migrations.



Websockets

Websockets protocal enables interaction between the client and the server of a web application over a single long living TCP connection with a low overheads, facilitating real-time data transfer from and to the server.




Pagination

will_paginate provides a simple API for performing paginated queries with Active Record, DataMapper and Sequel, and includes helpers for rendering pagination links in Rails, Sinatra and Merb web apps. We use "will_paginate-bootstrap4" and "will_paginate" gems for improving our webapp interface.


Ajax

The use of AJAX enables us to retrieve data for a web page without having to refresh the contents of the entire page. In our case, when user clicks a link or submits a form, The form is submitted to the server and the server returns an HTML fragment that contains the server's response and displays only the data that is new or changed as opposed to refreshing the entire page.

Rails-admin

We use RailsAdmin for data management. RailsAdmin is a Rails engine that provides an easy-to-use interface for managing data. In addition, we create an admin account, login with a specific account to check all user information.




Engineering Challenges

Uploaded profile pictures are displayed in different sizes

When users upload their profile pictures from local storage, the pictures usually come in different sizes which would break our site's layout. Unfortunately active storage doesn't have anything to avoid this. Therefore, we install imagemagick to resize these pictures before displaying.

When selecting a state, the candidate list of cities cannot change accordingly

We used ajax, action cable, tried to follow the gorails tutorial, and transformed it into our project. Applied in the dynamic drop-down menu, and real-time chat function.

Methodologies

Development Strategy

We adapts Agile's Feature-Driven Development Methodology. Namely we develop by features. We break whatever functions we are not able to finish implementing in a week into smaller and manageable features, and each member in our team is responsible for the performance and quality of each feature.

Deployment Principle

Automated deployment ensures us to ship code fully-automatically across all the stages of the development process. We enable automatic deploys for our github repo, under which situation all pushes are built and deployed to Heroku.In this way we significantly reduce the possibility of errors.


Testing Approach

We follow Agile's Incremental Functional Test Approach. As soon as a developer in our team finishes a feature, he or she marks this subtask as finished in Trello. After that a tester will pick this finished task and change it to "Under Testing". This way we can test the user stories in incremental fashion.
